# CODE_REVIEW_RULES.md

## 1. 리뷰 목적

이 프로젝트의 코드 리뷰는 모델 성능보다 **데이터 누수 방지, 재현성, 설명 가능성**을 우선한다.

## 2. 필수 확인 항목

### 데이터

- [ ] raw/interim/processed 경로가 분리되어 있는가?
- [ ] 원천 API 컬럼명이 표준 컬럼명으로 정규화되는가?
- [ ] 날짜 컬럼이 datetime으로 변환되고 정렬되는가?
- [ ] 종목 데이터와 지수 데이터 병합 방식이 명확한가?

### Label

- [ ] `future_return_5d`가 `close.shift(-5)` 방식으로 생성되는가?
- [ ] 마지막 5개 행이 학습에서 제외되는가?
- [ ] label 또는 future_return이 feature에 포함되지 않는가?

### Feature

- [ ] feature가 오늘까지 알 수 있는 데이터로만 계산되는가?
- [ ] rolling 계산의 window가 명확한가?
- [ ] 결측치 처리 방식이 기록되어 있는가?

### 모델

- [ ] train/validation/test가 날짜 기준으로 분할되는가?
- [ ] `train_test_split(..., shuffle=True)`를 쓰지 않는가?
- [ ] 모델 파일과 평가 결과가 저장되는가?

### 백테스트

- [ ] 모델 전략과 buy-and-hold가 분리되어 계산되는가?
- [ ] 거래비용이 파라미터로 분리되어 있는가?
- [ ] 백테스트 결과를 투자 추천처럼 표현하지 않는가?

## 3. 반려 기준

아래 중 하나라도 있으면 반려한다.

- API 키 하드코딩
- 랜덤 데이터 분할
- 미래 수익률을 feature로 사용
- 실시간 자동매매처럼 오해될 수 있는 문구
- 실행되지 않는 README 명령어
- reports 없이 결과만 출력
