# 프로젝트 기획서

## 1. 프로젝트 개요

본 프로젝트는 공공/공개 금융 데이터를 자동 수집하여 실제 주식의 향후 방향성을 예측하는 머신러닝 파이프라인을 구축하는 것을 목표로 한다. 특정 종목의 과거 시세, 거래량, 시장지표, 재무정보를 수집하고, 오늘 장 마감 시점까지 알 수 있는 데이터만 사용하여 향후 5영업일 수익률이 양수인지 예측한다.

## 2. 핵심 문제 정의

### 비즈니스/분석 질문

오늘 장 마감 시점까지 공개적으로 알 수 있는 데이터로, 해당 종목의 향후 5영업일 수익률이 0%보다 클지 예측할 수 있는가?

### 머신러닝 문제

- 문제 유형: 이진 분류
- 입력값 X: 일별 주가, 거래량, 거래대금, 이동평균, 변동성, 시장지수 수익률 등
- 예측값 y: 향후 5영업일 수익률이 0%보다 큰지 여부

```text
future_return_5d = close[t+5] / close[t] - 1
label = 1 if future_return_5d > 0 else 0
```

## 3. 프로젝트 포지셔닝

이 프로젝트는 “주가를 맞히는 AI”가 아니다. 핵심은 다음이다.

1. 실제 공개 금융 데이터를 수집한다.
2. 미래 데이터를 현재 예측에 섞지 않는다.
3. 시간 순서 기반으로 모델을 검증한다.
4. ML 지표와 투자 성과의 차이를 비교한다.
5. 모델의 한계와 개선 방향을 정직하게 문서화한다.

## 4. 데이터 범위

### 1차 데이터

| 데이터 | 목적 | 주요 컬럼 |
|---|---|---|
| 개별 종목 일별 시세 | 기본 가격/거래량 feature 생성 | 기준일자, 종목코드, 시가, 고가, 저가, 종가, 거래량, 거래대금, 시가총액 |
| 시장 지수 | 시장 전체 움직임 보정 | KOSPI/KOSDAQ 종가, 수익률, 변동성 |

### 2차 데이터

| 데이터 | 목적 | 주의사항 |
|---|---|---|
| 금융투자협회 시장자금 통계 | 유동성/수급 분위기 반영 | 주기와 발표일 확인 필요 |
| OpenDART 재무제표 | 기업 펀더멘털 반영 | 공시 제출일 이후부터 사용해야 함 |
| 한국은행 ECOS | 금리/환율/거시 변수 반영 | 발표 주기와 지연일 확인 필요 |

## 5. 1차 모델 범위

- 종목: 삼성전자 `005930`
- 데이터 기간: 2018-01-01 ~ 2025-12-31
- feature:
  - `return_1d`, `return_3d`, `return_5d`, `return_20d`
  - `ma_5`, `ma_20`, `ma_60`
  - `close_vs_ma5`, `close_vs_ma20`, `close_vs_ma60`
  - `volume_change_1d`, `volume_ma_5`, `volume_ma_20`
  - `volatility_5d`, `volatility_20d`
  - `market_return_1d`, `market_return_5d`
  - `excess_return_5d`
- label:
  - `label_up_5d`
- 모델:
  - LogisticRegression
  - RandomForestClassifier
- 평가:
  - accuracy, precision, recall, f1, roc-auc
  - confusion matrix
  - 단순 백테스트 수익률

## 6. 검증 전략

랜덤 분할은 사용하지 않는다.

기본 분할:

```text
train      : 2018-01-01 ~ 2023-12-31
validation : 2024-01-01 ~ 2024-12-31
test       : 2025-01-01 ~ 2025-12-31
```

추후 확장:

- walk-forward validation
- rolling window retraining
- 종목군별 out-of-sample 검증

## 7. 백테스트 정의

아래의 단순 전략부터 시작한다.

```text
모델이 상승이라고 예측한 날 종가 기준 매수
5영업일 후 종가 기준 매도
거래비용 차감
```

비교 기준:

- 모델 전략 평균 수익률
- buy-and-hold 수익률
- 거래 횟수
- 승률
- 최대 낙폭
- 거래비용 반영 후 수익률

## 8. 주요 리스크

| 리스크 | 설명 | 대응 |
|---|---|---|
| 데이터 누수 | 미래 가격/공시정보가 feature에 섞일 수 있음 | feature 생성 후 shift/label 검증 |
| 생존편향 | 현재 살아있는 종목만 쓰면 과거 성과가 과대평가될 수 있음 | 1차는 단일 종목으로 한정, 2차에서 설명 |

| 투자 성과 오해 | 정확도가 높아도 수익이 낮을 수 있음 | ML 평가와 백테스트 분리 |
| 재현성 부족 | 실행 환경/키/데이터 버전 차이 | README, `.env.example`, reports 작성 |

## 9. 성공 기준

1차 성공 기준은 “높은 수익률”이 아니라 다음이다.

- 실제 공개 API 기반 수집 코드가 있다.
- 학습용 데이터셋을 재생성할 수 있다.
- label이 미래 5영업일 기준으로 정확히 생성된다.
- 시간 순서 기반 평가가 가능하다.
- 모델 성능과 단순 백테스트를 분리해 설명할 수 있다.
- README와 보고서만 봐도 프로젝트 의도를 이해할 수 있다.

## 10. 블로그 메시지

이 프로젝트의 핵심 메시지는 다음이다.
